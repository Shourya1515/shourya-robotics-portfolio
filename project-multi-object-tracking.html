Multi-Object Tracking with Appearance-Based Re-Identification

Northeastern University · Jan 2026 – Apr 2026 · Python · PyTorch · OpenCV · YOLOv8

Final project for EECE 5639: Computer Vision, built collaboratively with Meet Brahmbhatt. Designed and implemented a complete multi-object tracking pipeline that combines YOLOv8 detection, Kalman prediction, Hungarian assignment, and ResNet18 appearance embeddings for re-identification through occlusions, re-entry, and cross-camera viewpoint changes.

Gallery

Overview

This project implements a multi-stage perception pipeline for pedestrian tracking and appearance-based re-identification. The system combines classical tracking methods with deep learning-based appearance embeddings, similar to the hybrid perception pipelines used in modern robotics and autonomous systems.

Level 1 — Single-Object Tracking

  • Used YOLOv8 for pedestrian detection from video frames.
  • Implemented constant-velocity Kalman filtering for motion prediction.
  • Performed IoU-based data association between predicted boxes and detections.
  • Achieved 95–97% mean IoU across custom indoor clips with under 0.3% frame loss.

Level 2 — Multi-Object Tracking with Re-ID

  • Extended tracking to multiple pedestrians using Hungarian algorithm assignment.
  • Built a combined IoU + cosine-distance cost matrix for matching detections to active tracks.
  • Used ResNet18 to generate 512-dimensional L2-normalized appearance embeddings.
  • Maintained a per-track appearance gallery to recover identities through occlusions and re-entry.

Level 3 — Cross-Camera Re-Identification

  • Extended the system to two synchronized cameras placed at a 90-degree offset.
  • Matched per-track gallery means between views using Hungarian assignment over cosine distance.
  • Achieved 100% identity match across two clip pairs, with 3/3 tracks matched per pair.
  • Maintained average cosine distance below 0.10 across correctly matched identities.

Results

  • 95–97% mean IoU on single-target tracking.
  • MOTA 0.50 and IDF1 0.71 on the MOT17-09-DPM benchmark.
  • Exceeded the SORT baseline by 16 MOTA points.
  • Achieved 100% cross-camera identity matching on a custom two-camera dataset.

Technical Implementation

  • Ported C++ embedding extraction utilities to Python / PyTorch.
  • Built modular detector, tracker, Re-ID, and evaluator components.
  • Tuned association weight, cosine-distance thresholds, and max-lost track lifetime.
  • Structured the system for repeatable evaluation and benchmark comparison.

Tech Stack

  • Python · PyTorch · OpenCV · YOLOv8 · ResNet18
  • SciPy · NumPy · Kalman Filter · Hungarian Algorithm
