NU Nance Report – Sensor Fusion & Localization

IMU · GPS · Complementary Filtering · Heading Estimation · Sensor Fusion

This project implements a complete localization pipeline using IMU and GPS data to estimate heading, forward velocity, and vehicle trajectory. The work focuses heavily on sensor fusion, calibration, and filtering techniques used in robotics and mobile navigation systems.

Gallery

Methodology

The localization pipeline was developed using IMU (accelerometer, gyroscope, magnetometer) and GPS data collected during structured driving sessions. All processing—including calibration, filtering, heading estimation, and trajectory reconstruction—was performed using robotics-inspired signal processing workflows.

  • Magnetometer Calibration: Hard-iron and soft-iron distortions were corrected by applying least-squares circle fitting to X–Y magnetometer data collected during circular driving. The calibration output was then applied to all future datasets.
  • Heading Estimation: Three parallel estimates were generated—magnetometer yaw, gyroscope-integrated yaw, and fused yaw—to evaluate sensor behavior and drift characteristics.
  • Complementary Filter: A Butterworth low-pass/high-pass split was used to fuse magnetometer stability with gyroscope responsiveness. The fused heading closely matched the ground-truth reference.
  • Velocity Estimation: Forward velocity was derived from integrated accelerometer data (after bias removal & low-pass filtering) and compared with GPS-differentiated velocity.
  • Trajectory Reconstruction: The fused heading and corrected forward velocity were used to generate a 2D dead-reckoning trajectory, highlighting challenges of pure IMU-based navigation.

Relevance to Robotics & ROS

The techniques used mirror common robotics localization workflows including IMU preprocessing, calibration pipelines, sensor fusion, heading estimation, and trajectory reconstruction. These methods extend directly to ROS environments using packages such as robot_localization, EKF/UKF filters, and SLAM tools that fuse IMU, wheel odometry, and GPS.

Key Takeaways

  • Calibration is essential for using low-cost IMUs in mobile robots.
  • Gyroscope drift makes long-term inertial navigation unreliable without fusion.
  • Complementary filtering provides a practical balance between noise rejection and responsiveness.
  • GPS is necessary for long-duration global track estimation.
  • IMU-only dead reckoning diverges rapidly without absolute corrections.
